WingIt

UX Designer

December 2012

WingIt was conceived and built in 8 hours during the 2013 API Hackday NY. This webapp is designed to connect people with similar interests to work on projects together. I worked mostly on visual design. In retrospect, the project was full of interaction design flaws, but I learned a lot from this project.

Visual Design

  • Home Page

    This is the first screen a user will see when they come across the Wingit website, which explains the purpose of the web app.

  • Event Discovery Page

    After signing up with WingIt, users can find events and projects happening around them on the “discover page.” For the sake of time, we had a predefined mile radius to define what was “local,” and did not implement a search feature. Circles in the right hand corner of each event card signify how many people are needed to make the event happen. Filled in circles represent filled slots, and empty circles represent empty slots for the projects/events.

  • Event Creation Page

    The “create” page allows a user to create a new event or project they would like people to join via WingIt. A giant interaction design flaw of the pictured system was that there was no way to have more or less than four attendees. A way to get around this would be to have only one blank “role here” section, and a plus sign that allowed users to make as many roles they wanted for the project.
